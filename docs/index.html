Feature-first .NET APIs with generated contracts.

1ファイル。 1機能。 API契約を生成。 One file. One feature. Generated API contracts.

標準のASP.NET Coreに近いまま、小さな.NET APIを1機能ずつ明示的に書き、routes・互換性・clientをあとから手で同期しないためのフレームワークです。 Build small .NET APIs explicitly, close to standard ASP.NET Core, without hand-syncing routes, compatibility notes, and clients later.

3つに絞った価値Three focused ideas

標準のASP.NET Coreに近いまま、feature fileと生成される契約をシンプルにつなぎます。Keep ASP.NET Core Minimal API close while connecting feature files with generated contracts.

Feature単位Feature-shaped APIs

1つの静的クラスが1つのエンドポイント。リクエスト、レスポンス、ハンドラー、バリデーションを近くに置けます。One static class maps to one endpoint, keeping request, response, handler, and validation close together.

生成されるAPI契約Generated API contracts

route metadata、互換性診断、typed clientを同じfeatureファイルから生成します。Generate route metadata, compatibility reports, and typed clients from the same feature files.

標準に沿うStandard by default

Minimal API binding、DI、endpoint filters、DataAnnotationsをそのまま使い、必要な時だけASP.NET固有のIResultに寄せられます。Keep Minimal API binding, DI, endpoint filters, and DataAnnotations; use ASP.NET-specific IResult only when a slice needs it.

Hello, Slice

リクエスト、レスポンス、ハンドラー、バリデーション。すべてを1つのファイルに。Request, response, handler, validation, and filters in one feature file.

Features/Users/CreateUser.cs
using System.ComponentModel.DataAnnotations;

namespace Slice.Sample.Features.Users;

[Feature("POST /users", Summary = "Create a new user")]
public static class CreateUser
{
    public record Request(
        [Required, MinLength(2)] string Name,
        [Required, EmailAddress] string Email);

    public record Response(Guid Id, string Name, string Email, DateTime CreatedAt);

    public static async Task<Response> Handle(Request req, IUserStore store, CancellationToken ct)
    {
        var user = await store.AddAsync(req.Name, req.Email, ct);

        return new Response(user.Id, user.Name, user.Email, user.CreatedAt);
    }
}
Program.cs
var builder = WebApplication.CreateSlimBuilder(args);

builder.Services.AddSlice();
builder.Services.AddSingleton<IUserStore, InMemoryUserStore>();

var app = builder.Build();

app.MapSlices();

app.Run();
