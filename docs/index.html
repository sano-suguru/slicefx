3つの思想の交差点Intersection of three ideas

Sliceは既存の優れたアイデアをASP.NET Core上に統合します。Slice combines proven ideas on top of ASP.NET Core.

FastEndpointsの構造FastEndpoints' shape

1つの静的クラスが1つのエンドポイントに対応。リクエスト、レスポンス、ハンドラー、バリデーションを近くに置けます。One static class maps to one endpoint, keeping request, response, handler, and validation close together.

FastAPIのDXFastAPI's DX

宣言的な属性、型駆動のバリデーション、パラメータベースのDI。Minimal APIのバインディングとOpenAPIをそのまま活かします。Declarative attributes, type-driven validation, parameter-based DI, and Minimal API binding/OpenAPI without hiding ASP.NET Core.

axumの規律axum's discipline

静的ハンドラー、生成コードによる登録、ホットパスのリフレクションゼロ。AOTとサーバーレスを第一級に扱います。Static handlers, generated registrations, and no hot-path reflection. AOT and serverless are first-class design targets.

Hello, Slice

リクエスト、レスポンス、ハンドラー、バリデーション。すべてを1つのファイルに。Request, response, handler, validation, and filters in one feature file.

Features/Users/CreateUser.cs
using System.ComponentModel.DataAnnotations;

namespace Slice.Sample.Features.Users;

[Feature("POST /users", Summary = "Create a new user")]
public static class CreateUser
{
    public record Request(
        [Required, MinLength(2)] string Name,
        [Required, EmailAddress] string Email);

    public record Response(Guid Id, string Name, string Email, DateTime CreatedAt);

    public static async Task<IResult> Handle(Request req, IUserStore store, CancellationToken ct)
    {
        var user = await store.AddAsync(req.Name, req.Email, ct);

        return Results.Created($"/users/{user.Id}",
            new Response(user.Id, user.Name, user.Email, user.CreatedAt));
    }
}
Program.cs
using Slice.Generated;

var builder = WebApplication.CreateSlimBuilder(args);

builder.Services.AddSliceGenerated();
builder.Services.AddSingleton<IUserStore, InMemoryUserStore>();

var app = builder.Build();

app.MapSlicesGenerated();

app.Run();
