機能ファイルからAPI契約を生成する.NET APIフレームワーク。Feature-first .NET APIs with generated contracts.

1ファイル。 1つの機能。 API契約は自動生成。 One file. One feature. Generated API contracts.

標準のASP.NET Coreに近い書き味で、小さな.NET APIを機能ごとに記述。ルート、互換性、クライアントの手作業同期を減らします。 Build small .NET APIs explicitly, close to standard ASP.NET Core, without hand-syncing routes, compatibility notes, and clients later.

大事にしている3つのことThree focused ideas

ASP.NET Core Minimal APIのよさを残しながら、機能ファイルと生成されるAPI契約を自然につなぎます。Keep ASP.NET Core Minimal API close while connecting feature files with generated contracts.

機能ごとに完結Feature-shaped APIs

1つの静的クラスが1つのエンドポイント。リクエスト、レスポンス、ハンドラー、バリデーションを同じ場所にまとめられます。One static class maps to one endpoint, keeping request, response, handler, and validation close together.

API契約を自動生成Generated API contracts

ルートメタデータ、互換性レポート、型付きクライアントを同じ機能ファイルから生成します。Generate route metadata, compatibility reports, and typed clients from the same feature files.

標準の作法をそのままStandard by default

Minimal APIのバインディング、DI、エンドポイントフィルター、DataAnnotationsをそのまま利用できます。ASP.NET固有のIResultが必要な場面にも対応します。Keep Minimal API binding, DI, endpoint filters, and DataAnnotations; use ASP.NET-specific IResult only when a slice needs it.

Hello, Slice

リクエスト、レスポンス、ハンドラー、バリデーション、フィルターまで。機能に関わるコードを1つのファイルにまとめられます。Request, response, handler, validation, and filters in one feature file.

Features/Users/CreateUser.cs
using System.ComponentModel.DataAnnotations;

namespace Slice.Sample.Features.Users;

[Feature("POST /users", Summary = "Create a new user")]
public static class CreateUser
{
    public record Request(
        [Required, MinLength(2)] string Name,
        [Required, EmailAddress] string Email);

    public record Response(Guid Id, string Name, string Email, DateTime CreatedAt);

    public static async Task<Response> Handle(Request req, IUserStore store, CancellationToken ct)
    {
        var user = await store.AddAsync(req.Name, req.Email, ct);

        return new Response(user.Id, user.Name, user.Email, user.CreatedAt);
    }
}
Program.cs
var builder = WebApplication.CreateSlimBuilder(args);

builder.Services.AddSlice();
builder.Services.AddSingleton<IUserStore, InMemoryUserStore>();

var app = builder.Build();

app.MapSlices();

app.Run();

コアは小さく、アダプタは必要な分だけSmall core, optional adapters

ホスティングやテストに必要な依存関係は、用途別のサテライトパッケージに分けています。Hosting and testing dependencies stay in satellite packages.

Slice.Core

属性、バリデーションフィルター、エンドポイントフィルターを含むコアランタイム。Core runtime: attributes, validation filters, and endpoint filters.

Slice.SourceGenerator

機能ルートの登録コードを、AOTに適した形で生成します。AOT-friendly generated registrations for feature routes.

Slice.Lambda

ASP.NET Core上でAWS Lambdaホスティングを利用するための連携パッケージ。AWS Lambda hosting integration over ASP.NET Core.

Slice.Workers

WorkersやWASIの実験向けに、ASP.NETに依存しないディスパッチを提供します。ASP.NET-independent dispatch for Workers and WASI experiments.

Slice.TestHost

SliceアプリをインプロセスでHTTPテストするためのヘルパー。In-process HTTP testing helpers for Slice apps.

Slice.Cli

sliceコマンドで、スキャフォールディング、ルート確認、互換性レポート、型付きクライアント生成を行えます。Scaffolding, route inspection, compatibility reports, and typed client generation through the slice command.
